# Outputs Directory

**Implementation author:** Dr. Jean-Jacques Ponciano

This directory stores generated results from the demonstration and verification workflows.

The expected generated files include:

- `demo_abdomen_bvtv.csv`: element-wise BV/TV and elastic properties using a fixed 300 HU threshold.
- `demo_abdomen_bvtv.materials.inp`: Abaqus material include file generated from `demo_abdomen_bvtv.csv`.
- `demo_abdomen_bvtv.mapped.inp`: self-contained Abaqus input combining the demonstration FE mesh with fixed-threshold BV/TV-derived material sections.
- `demo_abdomen_bvtv_otsu.csv`: element-wise BV/TV and elastic properties using Otsu thresholding.
- `demo_abdomen_bvtv_otsu.materials.inp`: Abaqus material include file generated from `demo_abdomen_bvtv_otsu.csv`.
- `demo_abdomen_bvtv_otsu.mapped.inp`: self-contained Abaqus input combining the demonstration FE mesh with Otsu-threshold BV/TV-derived material sections.
- `demo_abdomen_bvtv_registered.csv`: same demonstration mapping executed through the registration interface.
- `demo_abdomen_bvtv_registered.materials.inp`: Abaqus material include file generated from `demo_abdomen_bvtv_registered.csv`.
- `demo_abdomen_bvtv_registered.mapped.inp`: self-contained Abaqus input generated through the registration interface.
- `demo_peri_implant_bvtv.csv`: hollow-cylinder peri-implant BV/TV demonstration output.
- `visualization_300/report.html`: visual report for the fixed-threshold run, including final mapped FE mesh views.
- `visualization_300/assets/mapped_fe_bvtv_3d.png`: final mapped FE mesh colored by fixed-threshold `BV/TV`.
- `visualization_300/assets/mapped_fe_positive_bvtv_3d.png`: fixed-threshold FE elements with nonzero mapped `BV/TV`.
- `visualization_300/assets/mapped_fe_modulus_3d.png`: final mapped FE mesh colored by fixed-threshold Young's modulus.
- `visualization_otsu/report.html`: visual report for the Otsu-threshold run, including final mapped FE mesh views.
- `visualization_otsu/assets/mapped_fe_bvtv_3d.png`: final mapped FE mesh colored by Otsu-threshold `BV/TV`.
- `visualization_otsu/assets/mapped_fe_positive_bvtv_3d.png`: Otsu-threshold FE elements with nonzero mapped `BV/TV`.
- `visualization_otsu/assets/mapped_fe_modulus_3d.png`: final mapped FE mesh colored by Otsu-threshold Young's modulus.
- `verification_report.md`: automated verification report.

These files can be regenerated by running:

```bash
bash examples/run_demo.sh
```

High-quality offline demonstration outputs are generated under `outputs/high_quality/` by:

```bash
bash examples/run_high_quality_offline_demo.sh
```

The high-quality directory is intentionally ignored by Git because it can contain large generated Abaqus inputs and high-resolution visual assets.
